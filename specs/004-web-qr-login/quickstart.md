# Quickstart: 网页端咸鱼扫码登录

## 概述

本功能允许管理员在网页前端直接扫码登录咸鱼账号，无需手动复制Cookie。

## 前置要求

1. Docker 环境已启动
2. 已构建包含 Playwright 依赖的镜像

## 构建 Docker 镜像

```bash
cd xianyu-auto
docker-compose build backend
```

**注意**: 首次构建需要下载 Chromium 浏览器（约100MB），时间可能较长。

## 启动服务

```bash
docker-compose up -d
```

## 使用流程

### 1. 添加账号

1. 打开浏览器访问 http://localhost
2. 进入「账号管理」页面
3. 点击「添加账号」按钮
4. 输入账号昵称（如"闲鱼账号1"）
5. 点击确定

### 2. 扫码登录

1. 在账号列表中，点击「扫码登录」按钮
2. 弹出二维码对话框
3. 使用闲鱼APP扫描二维码
4. 在手机上确认登录
5. 等待自动完成登录，账号状态变为「在线」

### 3. 查看登录状态

- 「在线」: Cookie 有效
- 「离线」: 未登录
- 「已过期」: Cookie 已失效，需要重新登录
- 「错误」: 登录异常

### 4. 手动刷新状态

点击「刷新状态」按钮可立即检测账号登录状态。

### 5. 重新登录

当账号状态为「已过期」时，点击「扫码登录」重新授权。

## 定时任务

登录状态检测任务会自动执行：
- **间隔**: 每6小时
- **动作**: 检测所有账号Cookie有效性
- **通知**: 检测到过期账号时发送邮件通知

可在后台日志中查看检测结果。

## 常见问题

### 二维码无法显示

1. 检查后端日志是否有错误
2. 确认 Playwright 浏览器是否正确安装
3. 检查网络是否能访问淘宝登录页面

### 扫码后无反应

1. 检查手机网络是否正常
2. 确认二维码是否超时（有效期约5分钟）
3. 点击「取消」后重新尝试

### 登录状态检测失败

1. 检查Cookie是否完整
2. 确认咸鱼API是否有访问限制
3. 尝试手动刷新状态
