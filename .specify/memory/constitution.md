# 闲鱼自动管理系统 Constitution

## Core Principles

### I. 完成即检查

每完成一个需求（User Story）或任务（Task），必须立即进行自我检查和验证，确保无错误后再继续下一个任务。

- 每个Task完成后必须检查输出是否符合预期
- 验证任务执行过程中是否引入新的错误
- 发现问题立即修复，不留到后续阶段
- 记录检查结果和修复内容

### II. 小步提交

代码变更应该小步提交，每次提交完成一个独立的功能点。

- 每个Task对应一个提交
- 提交信息清晰描述完成的内容
- 保持代码库处于可工作状态

### III. 文档同步

代码变更必须同步更新相关文档。

- API变更需更新 contracts/
- 数据模型变更需更新 data-model.md
- 部署变更需更新 quickstart.md

## Development Workflow

### 任务执行流程

1. 从tasks.md中选取下一个任务
2. 执行任务内容
3. 立即进行自我检查：
   - 检查代码是否有语法错误
   - 检查功能是否符合预期
   - 检查是否引入新的问题
4. 如发现问题，立即修复
5. 验证修复成功后，继续下一个任务

### 检查清单

每个任务完成后必须验证：
- [ ] 代码能正常编译/构建
- [ ] 功能按预期工作
- [ ] 没有引入新的错误
- [ ] 相关文档已更新（如需要）
